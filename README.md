## 为什么值得看

| 指标 | 值 |
|------|-----|
| 公式量 | **940 条**（< 1000 预算） |
| 收尾一步平均步数 | **12.6** 步 |
| 凑段步数（精确最优） | 12–14 步 |
| 角空间最远距离 | 11 步（同二阶 God's number） |
| 全流程实例 | 37–44 步 |
| 收尾公式不含 F/B 90° | 只含 U D R2 L2 F2 B2 |

**与传统方法对比：**

| 方法 | 公式量 | 平均步数 | 背完需要 |
|------|--------|----------|----------|
| CFOP | ~120 | ~55 | 3–6 月 |
| Roux | ~500 | ~45 | 6–12 月 |
| ZB | ~800 | ~43 | 1–2 年 |
| **本法** | **~940** | **~37–44** | **纯直觉+背一条公式表** |

本方法的前两段**不需要背公式**——角段就是二阶魔方（已会），凑段靠直觉和几条原则。公式压力全在最后一段。

## 方法概览

```
┌─────────────────────────────────────────────────────┐
│  ① 角解  │  ② 凑棱  │  ③ 查表收尾                  │
│  ────────  │  ────────  │  ────────────                │
│  二阶直觉  │  守恒纪律  │  940 条公式，一眼识别       │
│  ~11 步    │  ~12-20 步 │  平均 12.6 步               │
│  0 公式    │  0 公式    │  需背诵                     │
└─────────────────────────────────────────────────────┘
```

### 阶段① — 角解（0 公式）
8 个角当二阶魔方解。唯一注意：三阶有固定中心色，角必须对齐中心。

### 阶段② — 凑棱（0 公式，直觉 + 三条原则）

**守恒纪律：** 拧的过程角会被打乱，只要你记两本账（面转账+切片账），查表前清零即可。

**三个子目标逐步推进：**

1. **中层归层** — FR/FL/BL/BR 都被中层棱占据
2. **清翻棱 (EO)** — 12 条棱朝向全对
3. **顶底分层** — 每层棱都在自己那层内

三个都达成 = 前置达成。清零账本，查表。

### 阶段③ — 一看收（940 条公式）

角全对 + 棱已分层 + EO 清零，此时魔方状态落在 **6912 种**可能性内。一层一层看棱的排列，唯一确定是哪条公式。

收尾公式全部只含 **U D R2 L2 F2 B2**（two-phase 的 phase-2 生成集），性质极好——不破坏棱定向、不扭角。拧错了中途停也只是位置乱，方向不乱。

## 3D 可视化演示

内置 Three.js 3D 魔方模拟器，支持键盘操作和分步执行。

### 启动
```bash
python3 server.py
# 浏览器打开 http://localhost:8000
```

或直接浏览器打开 `index.html`（需允许本地文件加载 Three.js CDN）。

### GUI 功能

## 数学基础

### 为什么选这个前置子集？

角解完后，剩余 4900 亿种状态。如果前置是 1000 个随机状态，需要在 9 步球（~10^8）里认出通往其中之一的路径——人不可能。

本方法的前置结构：

> **8 角全对 + 12 棱 EO=0 + 每条棱在自己那一层内**

这个子集 = `4! × 4! × 4! / 2 = **6912**` 种状态（总奇偶性被角锁定为偶）。

按 8 个观察视角折叠（y 转 ×4, z2 翻转 ×2），核心公式 **940 条**（864 自由轨道 + 76 对称特例）。

### 识别为什么友好

角全对 = 天然参照系。你只需要分别看三层（U/D/M），每层 4 条棱的排列模式唯一确定一条公式。**全程无 AUF**（转 U 会破坏角）。

### 公式长度分布（6912 态）

| 步数 | 条数 | 累计 |
|------|------|------|
| 6–9 | 126 | 126 |
| 10 | 221 | 347 |
| 11 | 520 | 867 |
| 12 | 2216 | 3083 |
| 13 | 2256 | 5339 |
| 14 | 1440 | 6779 |
| 15 | 132 | 6911 (+1 已解) |

**平均 12.6 步**。

## 变体：全查表流水线（v3）

零思考、零凑的纯查表变体（`gen_milestones.py` 生成）：

```
角(人肉~13)
  ↓ EO表(298条, 均8.3步)
  ↓ 中层表(72条, 均9.0步)
  ↓ 分层表(18条, 均6.9步)
  ↓ 终表(940条, 均12.6步)
```

- 总公式量：1328 条
- 平均总步数：~50
- 5 个 look，零思考
- 每张表按单一图案检索（翻色位置 / 中层棱位置 / 顶棱位置），识别极简单

## 技术细节

### 求解器核心
- **CubieCube**：魔方状态的高效表示（permutation + orientation 分离）
- **IDA***：角空间精确求解（掩码为二阶子空间）
- **Two-phase**：标准 Kociemba 算法，用于收尾阶段
- **Thistlethwaite 3**：三阶段降群解法，用于基准对比

### 坐标系统
- `coord.py`：flip（2048 种朝向）+ slice（495 种中层分布）+ corner perm / twist + edge8 + slice perm
- 各坐标分别对应 pruned table，加速 IDA* 搜索

### 生成与验证
```bash
python3 gen_onelook.py    # 生成 940 条收尾公式（含全量验证）
python3 rotsym.py         # 验证 8 个观察视角的等价性
python3 cou_tools.py      # M/E/S 定义验证 + 凑段基元搜索
python3 measure_cou2.py   # 全流程步数分布实测
```
